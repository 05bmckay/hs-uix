# v0.3.0

## Added

- Per-row action column via `rowActions` (static array or `(row) => actions[]`).
- `hideRowActionsWhenSelectionActive` option to hide per-row action column while selected-row action bar is visible.
- Text truncation helper via column `truncate`:
  - `truncate: true` for single-line truncation with tooltip.
  - `truncate: { maxLength: N }` for character-limit truncation with tooltip.
- Standard empty-cell fallback of `--` for `null`, `undefined`, and `""`.
- New edit types:
  - `time` (`TimeInput`, value `{ hours, minutes }`)
  - `datetime` (`DateInput + TimeInput`, value `{ date, time }`, with `editProps.timeProps`)
- Full-row edit mode via `editingRowId`, designed for Edit/Done flows.
- `scrollable` table mode (default `false`) to favor `"min"` fallback widths and horizontal overflow.
- Server-side selection callback `onSelectAllRequest({ selectedIds, pageIds, totalCount })`.
- Selection reset controls:
  - `resetSelectionOnQueryChange` (default `true`)
  - `selectionResetKey` for explicit parent-driven reset boundaries
- Live input callback `onRowEditInput(row, field, inputValue)` for draft validation/state.

## Changed

- In server-side mode, action-bar "Select all":
  - keeps page-level visual selection behavior
  - additionally supports dataset-level intent via `onSelectAllRequest`
- Sort contract normalized:
  - `sort` accepts `{ field, direction }` or `{ [field]: direction }`
  - `onParamsChange` emits `{ field, direction } | null`
- Select rendering behavior:
  - only discrete click-to-edit selects use `variant="transparent"`
  - inline/full-row selects use default bordered styling
- Date handling docs expanded in component docs/README.

## Fixed

- Date range filtering now excludes rows with invalid/unparsable date values.

## Migration Notes

- If you were checking for `null` in `onSortChange`, update to handle `"none"` when sort is cleared.
- For server-side tables that need global "select all matching", wire `onSelectAllRequest`.
- For full-row Edit/Done flows, use:
  - `editingRowId` to toggle row edit mode
  - `onRowEditInput` for live draft/validation
  - `onRowEdit` for committed field updates
